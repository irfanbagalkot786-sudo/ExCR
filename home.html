Introduction

What is this system?

The AI-Driven Classroom Monitoring Engagement System is a smart solution that uses Computer Vision and Artificial Intelligence to analyze students' behaviour in real time. It captures live video from a classroom, detects student faces, and classifies their engagement levels into four key states — then presents actionable insights through a web dashboard built with Django.

System Architecture

Core System Modules

Six tightly integrated modules form the backbone of the monitoring pipeline.

01
📷
Video Capture
OpenCV · CCTV / Webcam
Captures real-time video using webcam or CCTV cameras. Continuously feeds frames to the processing pipeline for immediate analysis at 30fps.
02
🧠
Face Detection
MediaPipe · Facial Landmarks
Uses MediaPipe's face mesh solution to detect and track 468 facial landmarks per student, enabling multi-face identification within a single frame.
03
📊
Behaviour Analysis
ML Classifier · Eye Tracking
Classifies each detected face into one of four engagement categories based on eye openness, head pose, and gaze direction.
04
⚡
Real-Time Processing
OpenCV · Frame Pipeline
Processes each frame in under 33ms to maintain real-time performance. Applies preprocessing, detection, classification, and annotation in a single optimized loop.
05
🌐
Web Dashboard
Django · REST API · WebSocket
A Django-powered web interface displays live engagement statistics, historical trends, class-level summaries, and exportable reports for teachers and administrators.
06
💾
Data Storage & Analytics
Django ORM · SQLite / PostgreSQL
All engagement records are persisted to a database with timestamps, enabling session-level analytics, trend analysis, and institutional reporting over time.

Engagement Categories

Student Behaviour Classification

The system recognises four distinct engagement states, each detected through specific facial and head-pose signals.

🧑‍🎓
Attentive
Student is focused, eyes open, and facing forward — actively engaged with the lecture or activity.
  • Eyes fully open
  • Head facing forward
  • Stable gaze direction
  • Upright posture
😴
Sleepy
Student shows signs of drowsiness — drooping eyelids, slow blinks, or a tilted head position.
  • Drooping eyelids (EAR low)
  • Slow blink rate
  • Head tilt detected
  • Yawning detection
👀
Distracted
Student's head or gaze is turned away from the board or instructor, indicating loss of focus.
  • Head yaw angle deviation
  • Gaze off-screen
  • Frequent head turns
  • Phone/object interaction
😶
Neutral
Student is present and awake but shows no clear signs of active engagement or distraction.
  • Eyes open, minimal focus
  • Low facial expressivity
  • Passive body language
  • No interaction signals

FAQ

Frequently Asked Questions

Common questions about the project, its technology, and real-world application.

01 What is this project about?
An AI-based system that monitors student engagement in classrooms in real time using computer vision and facial analysis.
02 Which technologies are used?
Python, OpenCV, MediaPipe, and Django form the core stack for video processing, face detection, and web-based result display.
03 What is the role of OpenCV?
OpenCV handles image processing, real-time video capture, frame preprocessing, and visual annotation of detection results.
04 Why MediaPipe for face detection?
MediaPipe provides fast, accurate, on-device face mesh detection with 468 facial landmarks — ideal for real-time engagement classification.
05 What are the four behaviour categories?
Attentive (focused), Sleepy (drowsy), Distracted (looking away), and Neutral (present but passive).
06 Is the system real-time?
Yes — the system processes live video frames continuously at up to 30fps, with minimal latency between capture and dashboard update.
07 What is Django used for?
Django serves as the web framework building the backend API, data models, admin interface, and live web dashboard for result display.
08 How does behaviour detection work?
By analysing facial landmarks — eye aspect ratio, head yaw/pitch angle, and gaze direction — to infer engagement state per student.
09 What are the limitations?
Performance may degrade in poor lighting conditions, with low camera quality, or where student privacy regulations restrict facial monitoring.
10 Can it work for online classes?
Yes — with integration to video conferencing platforms like Zoom or Meet, the system can extend monitoring to remote learning sessions.
